# 架构草图

当前后端骨架采用四层结构：

## 1. API 层

负责暴露 HTTP 接口，例如：

- 创建研究会话
- 获取当前会话状态
- 推进一次新的行动研究回合

## 2. Application 层

负责用例编排，不直接承载底层存储和模型调用细节。

典型职责：

- 创建会话
- 回读教师修订
- 调用优先级判断逻辑
- 组织提问与草稿生成流程

## 3. Domain 层

负责表达业务核心概念：

- 当前优先推进的研究部分
- 行动研究循环阶段
- 会话状态
- 优先推进规则

## 4. Infrastructure 层

负责对接外部依赖：

- LLM 客户端
- 数据库存储
- 仓储实现
- Prompt 组装

这种结构适合后续逐步扩展，而不需要把业务逻辑和接口、模型调用混在一起。
